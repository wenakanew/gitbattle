# Support

## Getting help

1. **Documentation:** Start with [**README.md**](README.md) and [**GitBattle_Project_Proposal.md**](GitBattle_Project_Proposal.md).
2. **Bug reports / feature ideas:** Open an [**issue**](https://github.com/wenakanew/gitbattle/issues) using the templates in `.github/ISSUE_TEMPLATE/`.
3. **How to contribute:** See [**CONTRIBUTING.md**](CONTRIBUTING.md).

---

## What we can help with

- Clarifying the specification or roadmap.
- Setup problems once the application codebase exists.
- Review of PRs that follow CONTRIBUTING guidelines.

---

## What to avoid

- **Security issues:** Do not file publicly — use [**SECURITY.md**](SECURITY.md).
- **Conduct concerns:** Follow [**CODE_OF_CONDUCT.md**](CODE_OF_CONDUCT.md) and contact maintainers as described in [**MAINTAINERS.md**](MAINTAINERS.md).

---

## Maintainer contact

See [**MAINTAINERS.md**](MAINTAINERS.md).

Response times are **best effort**; this is an independent open-source project.
