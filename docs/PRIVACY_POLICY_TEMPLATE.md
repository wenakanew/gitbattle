# Privacy policy (template)

**Status:** Template only — **not** a finished legal document. Replace placeholders and **have a lawyer review** before you operate GitBattle as a public service that collects personal data.

GitBattle will use **GitHub OAuth** and store account-related data as described in your technical specification. A typical privacy policy should cover:

1. **Data controller** — Who operates the service (your name or legal entity, Kenya address if applicable). *Draft contact you can adapt:* **Kaniu Jeffray Muthondu**, reach via **kaniujeffray@gmail.com**.
2. **What you collect** — e.g. GitHub user id, username, avatar URL, email if GitHub exposes it, usage logs, IP addresses (often in server logs).
3. **Why you process it** — authentication, matchmaking, leaderboards, security, legal compliance.
4. **Legal basis** — if you serve EU users: contract / legitimate interest / consent where required.
5. **Retention** — how long accounts, logs, and match history are kept.
6. **Sharing** — hosting providers (e.g. Railway, Vercel), analytics if any, **no selling** of personal data (state explicitly).
7. **International transfers** — if data leaves Kenya or users’ countries; standard contractual clauses if needed.
8. **Rights** — access, correction, deletion requests and how to contact you.
9. **Children** — whether the service is directed at minors (school use may matter).
10. **Changes** — how you notify users of policy updates.

Link this policy from your **README**, **footer**, and **OAuth app** configuration when you register the application with GitHub.
